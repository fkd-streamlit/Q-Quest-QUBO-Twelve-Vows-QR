## プロジェクト概要

**Q-Quest 量子神託**は、QUBO（Quadratic Unconstrained Binary Optimization）技術を用いて、日本の伝統的な「縁」の概念を量子技術で表現した価値体験プロジェクトです。

### ビジョン

- **Human-Centric Quantum Philosophy**という新しい潮流を、日本から世界へ発信
- 人と人とをつなぐ"縁"を量子技術が創造する
- 文化の懸け橋となり、次世代に向けた価値体験を提供
- 日本の文化を見直し、世界に向けて発信
- 海外の方が逆に日本を魅力と感じるポイントをしっかり見直して、活かす

## 機能（app_qubo.py）

### 主な機能

1. **テキスト入力からの自動ベクトル化**
   - ユーザーの状況を一文で入力
   - N-gram（3文字）とコサイン類似度で誓願ベクトルを自動推定
   - テキスト入力の影響を可視化

2. **手動スライダー入力**
   - 各誓願に対して0〜4のスライダーで手動調整可能
   - リアルタイムでmixベクトルに反映

3. **手動と自動の混合（mix）**
   - `alpha`パラメータで手動と自動の比率を調整
   - 0.0 = 手動のみ、1.0 = 自動のみ

4. **QUBO計算（One-Hot Encoding）**
   - One-Hot制約付きQUBOでキャラクターを選択
   - シミュレーテッドアニーリング（SA）によるサンプリング
   - エネルギー地形と観測分布の可視化

5. **観測結果の表示**
   - 観測されたキャラクター（神）の画像と情報
   - 寄与した誓願のTop表示（HTMLテーブルで白文字表示）
   - エネルギー地形（全候補）の表示
   - 観測分布（サンプル）の可視化

6. **神託の表示（2種類）**
   - **選ばれた神の神託（緑色）**: CHAR_MASTERの「役割」×寄与Top誓願から生成
   - **QUOTES神託（青色）**: 温度付きでQUOTESシートから選択

7. **キーワード抽出と単語の球体アート**
   - 入力テキストからキーワードを抽出
   - 3D球体で単語間の関係を可視化

### パラメータ調整

- **alpha（手動 vs 自動）**: 0.0〜1.0（デフォルト: 0.55）
- **格言温度（QUOTES選択のランダムさ）**: 0.2〜2.5（デフォルト: 1.2）
- **β（分布の尖り具合）**: 0.1〜10.0（デフォルト: 2.2）
- **サンプル回数**: 50〜2000（デフォルト: 300）
- **微小ゆらぎ ε**: 0.0〜0.30（デフォルト: 0.08）
- **ペナルティ P（One-Hot制約）**: 10.0〜100.0（デフォルト: 40.0）

## Excelファイルの形式

### VOW_DICT / VOW_MASTER（誓願辞書）

| 列名 | 説明 | 必須 |
|------|------|------|
| VOW_ID | 誓願ID | ✅ |
| TITLE | 誓願名 | ✅ |
| DESCRIPTION_LONG | 説明（長文） | ⚠️ 推奨 |
| DESCRIPTION | 説明 | ⚠️ 代替 |

### CHAR_MASTER（キャラクター情報）

| 列名 | 説明 | 必須 |
|------|------|------|
| CHAR_ID | キャラクターID | ✅ |
| 公式キャラ名 | キャラクター名 | ✅ |
| 役割 | キャラクターの役割 | ⚠️ 推奨（神託生成に使用） |
| 役割補足説明 | 役割の補足説明 | ⚠️ 任意（神託生成に使用） |
| 属性 | キャラクターの属性 | ⚠️ 任意 |
| 絵馬文字 | 絵馬の文字 | ⚠️ 任意 |
| 分析 | 分析情報 | ⚠️ 任意 |

### CHAR_TO_VOW（キャラクター×誓願の重み行列）

| 列名 | 説明 | 必須 |
|------|------|------|
| CHAR_ID | キャラクターID | ✅ |
| IMAGE_FILE | 画像ファイル名 | ⚠️ 推奨 |
| VOW_01, VOW_02, ... | 各誓願への重み（数値） | ✅ |

### STAGE_TO_VOW（季節×時間の状態、任意）

| 列名 | 説明 | 必須 |
|------|------|------|
| STAGE_ID | ステージID | ✅ |
| VOW_01, VOW_02, ... | 各誓願への重み（数値） | ✅ |

### QUOTES（格言データ、任意）⭐

QUOTESシートは、温度付きで格言を選択する機能で使用されます。140件近い格言を登録することで、より豊かな神託体験を提供できます。

| 列名 | 説明 | 必須 | 備考 |
|------|------|------|------|
| QUOTE | 格言テキスト（日本語） | ✅ | 必須。空の行は自動的に除外されます |
| QUOTE_ID | 格言ID（例：Q_0001, Q_0002） | ⚠️ 推奨 | ない場合は自動生成されます |
| SOURCE | 出典・著者名（日本語） | ⚠️ 推奨 | ない場合は空文字列になります |
| LANG | 言語コード（通常は"ja"） | ⚠️ 推奨 | ない場合は自動的に"ja"が設定されます |

**QUOTESシートの検索:**
- シート名は柔軟に検索されます（"QUOTES", "QUOTE", "格言", "格言一覧"など）
- 大文字小文字は区別されません
- シート名に空白が含まれていても検出されます

**温度付き選択:**
- 温度スライダー（0.2〜2.5）で格言の選択方法を調整できます
- 温度が高いほどランダムに選択されます
- 温度が低いほど上位の格言が固定されます
- 温度を変更すると、異なる格言が表示されます

## 技術的な詳細

### QUBOモデル（One-Hot Encoding）

エネルギー関数: `E(x) = Σ_i (-score_i) x_i + P*(Σ_i x_i - 1)^2`

- **score_i**: キャラクターiのスコア（`W × mix_vec`）
- **x_i**: バイナリ変数（0 or 1）
- **P**: One-Hot制約のペナルティ（1つだけ選ぶ）
- **W**: キャラクター×誓願の重み行列（CHAR_TO_VOWから読み込み）
- **mix_vec**: 手動と自動の混合ベクトル

### テキストからの自動ベクトル化

1. **N-gram抽出**: 入力テキストと各誓願の説明文から3文字N-gramを抽出
2. **コサイン類似度計算**: N-gramベクトル間のコサイン類似度を計算
3. **正規化**: 最大値で正規化し、0〜5のスケールに変換

### シミュレーテッドアニーリング（SA）

- One-Hot制約を維持しながら、エネルギー最小化を探索
- βパラメータで温度を制御
- 複数回サンプリングして分布を生成

### 神託生成

1. **選ばれた神の神託（緑色）**:
   - CHAR_MASTERの「役割」列を取得
   - 「役割補足説明」があれば結合
   - 寄与Topの誓願タイトルを追加
   - Stage情報を追加

2. **QUOTES神託（青色）**:
   - QUOTESシートから言語フィルタリング
   - 温度パラメータでソフトマックス分布を計算
   - 確率的に選択

## トラブルシューティング

### よくある問題

1. **Excelファイルが読み込めない**
   - シート名が `VOW_DICT`/`VOW_MASTER`, `CHAR_MASTER`, `CHAR_TO_VOW` であることを確認
   - 必須列が存在することを確認
   - ファイル形式が `.xlsx` であることを確認

2. **画像が表示されない**
   - 画像フォルダのパスを確認
   - `CHAR_TO_VOW` の `IMAGE_FILE` 列に正しいファイル名が記載されているか確認
   - ファイル名に `CHAR_01` や `1` などが含まれているか確認

3. **表の文字が黒く見える**
   - ブラウザのキャッシュをクリア（Ctrl+F5）
   - アプリを再起動

4. **QUBO計算が動作しない**
   - `app_qubo.py` を使用していることを確認
   - 統合Excelが正しく読み込まれているか確認
   - 必須シートが存在することを確認
